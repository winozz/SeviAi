# DIWA / SeviAi — Knowledge Transfer (KT) Document

**Audience:** A new developer or maintainer who has never touched this project.
**Goal:** After reading this, you can run the bot, change what it says, retrain it, and avoid the common mistakes.

---

## 1. What this project is, in one paragraph

**DIWA** ("Digital Interface Web Assistant") is a chatbot for **Cavite State University (CvSU)** that answers student questions about admissions, programs, tuition, scholarships, campus life, etc. It's built as a Python ML pipeline (Naive Bayes + a small neural network) on top of a JSON "intents" file. The chatbot is served by a FastAPI web server and has two frontends: a simple HTML page (`web/`) and a Next.js web app (`SeviWeb/`).

```
User → FastAPI (api/app.py) → Hybrid classifier → Response from intents.json
                                    │
                                    ├─ Naive Bayes (fast, confident classifier)
                                    └─ Neural Network (fallback for ambiguous input)
```

## 3. How the data flows (memorize this)

```
data/cavsu_intents.json  ← You edit this
        │
        ▼
intents_db.py            ← Auto-syncs the SQLite DB if JSON changed
        │
        ▼
data/cavsu_intents.db    ← SQLite cache, used at runtime
        │
        ▼
train_naive_bayes.py     ← Reads DB, trains, writes:
train_hybrid.py            └── models/cavsu_classifier.pkl
                           └── models/nn_model.h5 + helpers
                           └── models/responses_map.json
        │
        ▼
api/app.py (the server)  ← Loads the trained models on startup
        │
        ▼
User gets an answer
```

**Critical rule:** editing the JSON ≠ the bot learns. You must retrain AND restart the server.

## 6. How to teach the bot something new

### Step 1: Edit the intents file

Open [SeviAi/data/cavsu_intents.json](../data/cavsu_intents.json). Each intent looks like:

```json
{
  "tag": "tuition_fees",
  "patterns": [
    "How much is tuition?",
    "Magkano ang tuition?",
    "What does it cost?"
  ],
  "responses": [
    "Under RA 10931, tuition is free for qualified Filipino undergrads at SUCs including CvSU. For exact fees, please check www.cvsu.edu.ph."
  ]
}
```

- **`tag`**: unique short name (snake_case).
- **`patterns`**: examples of how users might ask. More variety = better matching. Mix English + Tagalog.
- **`responses`**: what the bot says. Multiple entries → the bot picks one at random.

**To add a new topic:** copy an existing intent block, change the `tag`, replace `patterns` and `responses`.
**To extend an existing topic:** just append to `patterns` or `responses`.

### Step 2: Retrain

```powershell
cd c:\Users\Admin\Documents\Sev\SeviAi
py -3.11 train_naive_bayes.py
py -3.11 train_hybrid.py
```

`intents_db.py` will auto-rebuild the SQLite DB from your edited JSON.

### Step 3: Restart the server

`Ctrl+C` the running server, then start it again. The old in-memory models won't reload by themselves.

### Step 4: Test

```powershell
py -3.11 test_chatbot.py
```

Or use the browser UI and ask the new question.

## 7. ⚠️ Content rules (read this before editing intents)

The bot must never train on **fabricated facts**. This was a real problem in this project — the dataset previously claimed "13 campuses" and "14 campuses" in different places, listed a Ternate campus that doesn't exist, invented student-population numbers, and quoted phone numbers that conflicted between files. All of that was sanitized.

### What you can safely write
- Verifiable legal/historical facts (RA 10931, RA 8468, founding year 1906, motto "Truth, Excellence, Service").
- Process descriptions that are stable (admission steps, registrar workflow).
- General categories ("CvSU offers programs across Agriculture, Engineering, IT, Education…").
- Pointers to authoritative sources (`www.cvsu.edu.ph`, `admission.cvsu.edu.ph`).

### What you must NOT write
- Specific campus counts ("13 campuses", "14 campuses"). Use "multiple campuses across Cavite".
- Specific lists of campus municipalities — these change as the university grows.
- Specific enrollment figures, faculty counts, library volumes.
- Specific phone numbers (they change; the website is authoritative).
- Specific fees in pesos (these change yearly).
- Specific calendar dates (Foundation Day, graduation, semester start).
- Anything you can't verify on cvsu.edu.ph today.

When in doubt: **state the general framework, then point to www.cvsu.edu.ph.**

This rule applies to:
- `data/cavsu_intents.json`
- `data/cavsu_intents_expanded.json`
- `patch_intents.py` (it bulk-writes responses)
- `research_feed.py` (it has reference "correct" answers)
- `api/campus_places.py` (directory module)

## 8. The hybrid classifier — quick mental model

`hybrid_chatbot.py` uses two models in sequence:

1. **Naive Bayes** runs first. If it's confident (above its threshold), use its answer. **Fast.**
2. If NB isn't confident, **Neural Network** takes over. It has per-intent thresholds calibrated during training.
3. If neither is confident, return the **`nlu_fallback`** response ("I'm not sure I understand…").

This is why both `train_naive_bayes.py` AND `train_hybrid.py` need to run after edits.

## 11. Troubleshooting

### "ModuleNotFoundError: No module named 'numpy'"
You skipped `pip install`. Run:
```powershell
py -3.11 -m pip install -r deployment\requirements_minimal.txt
```

### "Port 8000 already in use"
Either kill the previous server or use a different port:
```powershell
py -3.11 -m uvicorn api.app:app --port 8001
```

### Bot answers with old info after I edited the JSON
You forgot to retrain OR forgot to restart the server. Do both:
```powershell
py -3.11 train_naive_bayes.py
py -3.11 train_hybrid.py
# Then Ctrl+C the server window and restart it.
```

### "JSON decode error" on startup
You have a syntax error in `cavsu_intents.json` — a missing comma, an unescaped quote, etc. Open it in VS Code; broken JSON shows up underlined in red.

### Bot keeps falling back to "I'm not sure I understand…"
Either:
- The new patterns don't cover the user's phrasing → add more variations.
- The neural net thresholds are too strict → check `models/nn_thresholds.json` after retraining.

### Trained DB and JSON are out of sync
They auto-sync via `intents_db.py:load_intents()`. To force a rebuild:
```powershell
py -3.11 -c "from intents_db import create_intents_database; create_intents_database(json_path='data/cavsu_intents.json', db_path='data/cavsu_intents.db', recreate=True)"
```

### "The bot quoted a wrong phone number / wrong campus list"
Search for the offending string across the active data files:
```powershell
py -3.11 -c "import json; d=json.load(open('data/cavsu_intents.json',encoding='utf-8')); [print(i['tag']) for i in d['intents'] for r in i['responses'] if 'OFFENDING_TEXT' in r]"
```
Then fix it in the JSON, retrain, and restart.

## 15. The five things that bite new devs

1. **Edit JSON without retraining** → bot doesn't change. Always retrain + restart.
2. **Restart-but-not-retrain** → old model loaded, ignores your edits.
3. **Run `patch_intents.py` without reading it** → it bulk-rewrites responses. Read §7 first.
4. **Trust hardcoded contact info** → always defer to `www.cvsu.edu.ph` for phones/emails/dates.
5. **Add a new intent but forget `"active": true`** (only matters for certain intents — most don't have this field; check existing entries before copying).
